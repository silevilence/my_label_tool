# 🗺️ Product Roadmap

## 📝 计划中 (Planned)

## 🚧 开发中 (In Progress)

- [ ] **实现标签配置面板 UI**
  - [ ] 新建 `components/settings/LabelSettings.tsx`，展示当前所有 `LabelConfig` 列表，替代此前硬编码的标签
  - [ ] 支持新增标签（输入名称，自动分配一个默认颜色）
  - [ ] 支持删除标签（若该标签已被使用中的矩形框引用，删除前需提示）
  - [ ] 支持修改标签颜色（用颜色选择器）

- [ ] **标签配置持久化到本地文件**
  - [ ] Rust 端新增 command：读取/写入标签配置到本地 JSON 文件（如放在应用数据目录）
  - [ ] 应用启动时自动加载上次保存的标签配置
  - [ ] 标签配置面板中的增删改操作实时同步保存到本地文件

- [ ] **实现标签快捷键绑定**
  - [ ] 在标签配置面板中，每个标签可设置一个单键快捷键（如数字键、字母键）
  - [ ] 校验新绑定的快捷键是否与其他标签冲突，冲突时提示并阻止保存
  - [ ] 按下对应快捷键时，当前选中的矩形框标签切换为该快捷键对应标签

- [ ] **预置常用标签模板**
  - [ ] 设计 1-2 套预置标签模板（如通用目标检测常见类别），存放在 `src/lib/defaults/`
  - [ ] 标签配置面板中提供“加载预置模板”按钮，可一键导入（需处理与现有标签的合并/覆盖逻辑）

- [ ] **设计导出模板数据结构与 UI 骨架**
  - [ ] 在 `src/types/export.ts` 中定义 `ExportTemplate` 类型
  - [ ] 新建导出面板 UI，展示可选导出格式列表（先只列出，不实现具体转换逻辑）

- [ ] **实现 COCO 格式导出**
  - [ ] 在 `src/lib/exporters/coco.ts` 中实现内部标注数据 → COCO JSON 结构的转换
  - [ ] Rust 端或前端完成文件写入（复用此前 JSON 导出流程的 command，调整输入数据结构）
  - [ ] 用 `pycocotools`（或在线校验工具）验证导出文件格式合法

- [ ] **实现 VOC 格式导出**
  - [ ] 在 `src/lib/exporters/voc.ts` 中实现内部标注数据 → VOC XML 结构的转换（每张图一个 XML 文件）
  - [ ] 验证导出的 XML 文件字段完整（bndbox、name、size 等）且能被常见 VOC 解析工具读取

- [ ] **实现 YOLO 格式导出**
  - [ ] 在 `src/lib/exporters/yolo.ts` 中实现内部标注数据 → YOLO txt 格式（归一化中心点+宽高）的转换
  - [ ] 同时生成对应的 `classes.txt` 标签索引文件
  - [ ] 验证坐标归一化计算正确（抽样人工核对至少3个标注框）

- [ ] **实现自定义导出格式（字段映射）**
  - [ ] 设计一个简单的字段映射配置 UI（如 JSON 结构模板，用户可指定字段名对应关系）
  - [ ] 在 `src/lib/exporters/custom.ts` 中实现基于该映射配置的通用转换逻辑
  - [ ] 用一个实际自定义模板跑通一次完整导出流程

- [ ] **实现画布缩放（滚轮缩放 + Ctrl 微调）**
  - [ ] 监听画布 `wheel` 事件，计算缩放前后鼠标指向的图片坐标点保持不变
  - [ ] 设置合理的最小/最大缩放级别限制
  - [ ] 区分普通滚轮与 Ctrl+滚轮两种缩放步长（Ctrl 时更细粒度），验证两种模式切换流畅、无跳变感
  - [ ] 验证缩放后已有标注框仍与图片内容对齐（无偏移）

- [ ] **实现快捷键切换图片**
  - [ ] 绑定默认快捷键（如 `A`/`D` 或方向键）切换上一张/下一张图片
  - [ ] 切图时正确保存当前图片标注状态、加载目标图片标注状态（复用已有的 annotation store 逻辑）
  - [ ] 图片列表面板同步高亮当前选中项，并自动滚动到可视区域

- [ ] **实现快捷键速切当前标签**
  - [ ] 按下标签绑定的快捷键时，若当前有选中的矩形框，直接切换其标签
  - [ ] 若当前无选中框，则切换“下一个新建矩形框默认使用”的标签
  - [ ] 界面上有明显指示当前默认标签是什么（如工具栏高亮）

- [ ] **实现通用快捷键配置面板**
  - [ ] 新建 `components/settings/ShortcutSettings.tsx`，列出所有可配置动作（缩放、切图、删除框、撤销/重做等）及当前绑定键
  - [ ] 支持点击后进入“录制”状态，按下新键位完成改绑
  - [ ] 改绑后的快捷键持久化保存到本地配置文件，应用重启后生效
  - [ ] 校验新绑定是否与已有快捷键冲突

- [ ] **实现撤销/重做栈**
  - [ ] 设计基于操作记录（新增/删除/移动/调整标注框）的撤销栈数据结构
  - [ ] 绑定默认快捷键（如 `Ctrl+Z` / `Ctrl+Y`）触发撤销/重做
  - [ ] 验证连续多步操作后撤销顺序正确，重做后状态与撤销前一致

- [ ] **实现删除标注框与批量清空**
  - [ ] 选中矩形框后按 `Delete` 键删除，操作计入撤销栈
  - [ ] 提供“清空当前图片所有标注”按钮（需二次确认，避免误触）

- [ ] **优化画布性能与大图片处理**
  - [ ] 测试较大分辨率图片（如 4K 以上）加载与标注时的流畅度
  - [ ] 若有明显卡顿，评估是否需要对图片进行降采样展示、标注框坐标仍按原图分辨率存储

- [ ] **完成 Windows 打包并在无开发环境机器上验证**
  - [ ] 执行 `npm run tauri build`，确认生成 Windows 安装包/exe 无报错
  - [ ] 在开发机器上运行打包产物，验证功能与 dev 模式一致
  - [ ] 找一台未安装 Node/Rust/前端开发环境的 Windows 机器（或全新虚拟机），安装/运行打包产物，记录启动耗时、是否有依赖缺失报错

- [ ] **优化打包体积与启动速度（若上一任务发现问题）**
  - [ ] 检查 `tauri.conf.json` 中是否有可裁剪的不必要功能模块
  - [ ] 检查前端打包产物是否有可优化的冗余依赖（如未使用的图标库、polyfill）
  - [ ] 记录优化前后的体积/启动耗时对比

- [ ] **编写打包与发布说明文档**
  - [ ] 在 `docs/` 下新增 `build.md`，记录打包命令、注意事项、已知问题
  - [ ] 记录支持的最低 Windows 版本（如需要）

- [ ] **实现多边形标注**
  - [ ] 支持连续点击添加多边形顶点，双击或按 Enter 完成绘制
  - [ ] 已完成的多边形可选中，顶点可单独拖拽调整
  - [ ] 更新 `coco`/`custom` 导出逻辑以支持多边形数据（VOC/YOLO 视格式本身是否支持决定是否跳过）

- [ ] **实现关键点标注**
  - [ ] 支持在图片上单击放置关键点，绑定标签
  - [ ] 关键点可选中并拖拽调整位置
  - [ ] 若需要关键点分组（如人体骨架点位关联），设计对应的数据结构扩展

- [ ] **实现图形类型工具栏切换与对应快捷键**
  - [ ] 工具栏增加图形类型切换按钮（矩形/多边形/关键点）
  - [ ] 绑定快捷键快速切换绘制模式

- [ ] **扩展标签配置以支持指定图形类型**
  - [ ] `LabelConfig.shapeType` 生效：不同标签仅能用于对应图形类型（或允许通用，视产品决策）
  - [ ] 标签配置面板 UI 相应调整，创建标签时可选择适用图形类型

- [ ] **实现视频文件导入与抽帧**
  - [ ] Rust 端接入视频处理能力（ffmpeg sidecar 或对应 crate），实现按固定间隔抽帧
  - [ ] 前端展示抽帧后的帧列表（类比此前的图片列表加载逻辑）

- [ ] **实现视频时间轴 UI**
  - [ ] 新建时间轴组件，展示当前视频总帧数、当前帧位置
  - [ ] 支持拖动时间轴快速跳转到指定帧

- [ ] **复用图片标注画布逻辑标注视频单帧**
  - [ ] 确认 `AnnotationShape.frameIndex` 字段正确关联每一帧的标注数据
  - [ ] 验证切换帧时标注加载/保存逻辑与图片标注一致

- [ ] **实现跨帧标注插值（可选，视需求优先级）**
  - [ ] 设计关键帧标注 + 中间帧线性插值的数据结构
  - [ ] 实现插值计算逻辑与对应 UI（标记关键帧、查看插值结果）

- [ ] **扩展导出格式以支持视频标注结果**
  - [ ] 评估现有导出格式（COCO/VOC/YOLO）是否有视频标注扩展惯例，或设计自定义结构
  - [ ] 实现对应导出逻辑并验证
